### ⚠️ ОБНАРУЖЕНА ПРОБЛЕМА — PROBLEM #1: Whisper не выгружается автоматически

**Место**: `window.py` → `_on_transcribe_done()`

**Проблема**: После транскрипции Whisper-модель остаётся в памяти навсегда.
В `_on_transcribe_done()` нет вызова `self.whisper_engine.unload_model()`.

**Проявление**: После транскрипции VRAM не освобождается, при последующей генерации TTS
может не хватить памяти → `OutOfMemoryError` или артефакты.

**Решение**: Добавить вызов `self.whisper_engine.unload_model()` в `_on_transcribe_done()`.

## 6. ⚠️ МОДУЛЬ СИНТЕЗА — ПОТЕНЦИАЛЬНЫЕ ПРОБЛЕМЫ

### PROBLEM #2: Потенциальные артефакты при склейке сегментов

**Место**: `ttS_engine.py` → `generate()`

**Проблема**: Аудио сегментов склеивается через `np.concatenate()` с 0.15с тишиной между ними:

```python
pause_samples = int(0.15 * sr_final)
pause = np.zeros(pause_samples, dtype=np.float32)
final_audio = audio_parts[0]
for part in audio_parts[1:]:
    final_audio = np.concatenate([final_audio, pause, part])
```

**Возможные артефакты**:
1. **Щелчки** при резком переходе от placeholder-аудио ("ха-ха") к нормальному тексту
   — 0.15с тишины помогает, но не всегда достаточно
2. **Срезание начала аудио** — если последний сегмент обрезается из-за паузы

**Вероятность артефактов**: НИЗКАЯ (0.15с тишины обычно достаточно)

**Рекомендация**: Добавить fade-in/fade-out на границах сегментов для плавности.

### PROBLEM #3: Crop расчёт в `generate_voice_clone()` (МОДЕЛЬ)

**Место**: `qwen3_tts_model.py` → `generate_voice_clone()`

```python
# Декодируем с референсом:
codes_for_decode.append(torch.cat([ref_code_list[i], codes], dim=0))

# Затем обрезаем:
ref_len = int(ref_code_list[i].shape[0])
total_len = int(codes_for_decode[i].shape[0])
cut = int(ref_len / max(total_len, 1) * wav.shape[0])
wavs_out.append(wav[cut:])
```

**Анализ**:
- `ref_len` = количество токенов референса
- `total_len` = ref_len + сгенерированные токены
- `ref_len / total_len` = доля референса в общем
- `cut` = сколько сэмплов отрезать от начала

**Корректность**: Формула корректна. Если ref_len=128, total=384 → cut = wav_len/3.
Отрезаем 1/3 начала (референс), оставляем 2/3 (сгенерированная речь).

**Граничный случай**: Если ref_len = 0 (x_vector_only_mode) → cut = 0 → весь wav.
Но в x_vector_only_mode ref_code = None, так что этот блок не выполняется.

**Статус: ✅ ПРОЙДЕНО** — формула корректна для всех случаев.

### PROBLEM #4: Обработка неизвестных инструкций

**Место**: `tts_engine.py` → `_find_similar_instruction()`

**Проблема**: Функция ищет частичные совпадения:
```python
if lower in known or known in lower:
    return known
```

**Сценарий**: Пользователь ввёл `[смейтся]` (опечатка вместо `[смеётся]`).
`_find_similar_instruction` проверит `if "смейтся" in "смеётся"` → False
и `if "смеётся" in "смейтся"` → False → инструкция будет пропущена.

**Решение**: Добавить fuzzy matching (Levenshtein distance) или более
расширенный словарь синонимов.

**Вероятность проблемы**: НИЗКАЯ (пользователь увидит предупреждение в логе)

### ⚠️ PROBLEM #5: Безопасность — `weights_only=False` в калибровке

**Место**: `window.py` → `_load_calibrate_voice_info()`, `_start_calibrate()`

```python
vector = torch.load(path, map_location="cpu", weights_only=False)
data = torch.load(path, map_location="cpu", weights_only=False)
```

**Проблема**: `weights_only=False` позволяет выполнение произвольного кода при
загрузке `.pt` файла. Если пользователь откроет malicious `.pt` файл, это может
привести к выполнению вредоносного кода.

**В TTS engine используется `weights_only=True`** → это правильный подход.

**Решение**: Заменить `weights_only=False` на `weights_only=True` в калибровке.
